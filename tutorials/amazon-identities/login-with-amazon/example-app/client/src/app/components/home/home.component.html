Login with Amazon (LWA) Integration Demo

This application demonstrates the OAuth 2.0 authorization flows for connecting to Amazon's three primary API surfaces via Login with Amazon (LWA).

Amazon Advertising API

Campaign management, reporting, bid optimization. Uses 3 regional OAuth URLs (NA, EU, FE).

Auth type: Public OAuth only

Connect Advertising

Seller Central (SP-API)

Orders, inventory, fulfillment, listings. Marketplace-specific authorization URLs.

Auth types: Public OAuth + Private App

Connect Seller

Vendor Central (SP-API)

Purchase orders, shipments, invoices. Unique URL per marketplace.

Auth types: Public OAuth + Private App

Connect Vendor

How It Works

Public Apps: Use OAuth consent flow — user is redirected to Amazon, authorizes, and returns with an authorization code exchanged server-side for tokens.
Private Apps (Seller/Vendor only): Self-authorized in Seller/Vendor Central. User provides their own Client ID, Client Secret, and Refresh Token at runtime. Each private app has its own credentials per account/marketplace.
